## What to Build

### 1. Wrangler Configuration

Add D1 and R2 bindings to your `wrangler.toml` (or `wrangler.jsonc`):

```toml
[[d1_databases]]
binding = "DB"
database_name = "cybiqon-blog"
database_id = "0a9bcd34-fd85-4972-9e5e-ec86dfe5ebab"

[[r2_buckets]]
binding = "BLOG_IMAGES"
bucket_name = "cybiqon-blog"
```

This gives your Next.js app access to D1 and R2 via Cloudflare Workers bindings (no API calls needed at runtime вҖ” direct binding is faster and free).

### 2. Blog List Page вҖ” `/blog`

**Route:** `src/app/blog/page.tsx`

**What it does:**
- Queries D1: `SELECT id, slug, title, excerpt, image_url, tags, created_at FROM blog_posts WHERE published = 1 ORDER BY created_at DESC`
- Renders a grid/list of blog post cards
- Each card shows: image, title, excerpt, date, tags
- Links to `/blog/[slug]`

**Design guidelines:**
- Brand colors: Blue `#3b82f6`, Orange `#f97316`, Dark `#0f172a`, White `#ffffff`
- Fonts: Poppins (headings), Inter (body)
- Clean, minimal layout вҖ” card grid (2-3 columns on desktop, single column mobile)
- Each card: image on top, title, excerpt (2 lines max), date, tag pills

**D1 access pattern (Cloudflare Pages):**
```typescript
// In your page or API route
export async function loader({ context }) {
  const db = context.cloudflare.env.DB;
  const { results } = await db.prepare(
    "SELECT id, slug, title, excerpt, image_url, tags, created_at FROM blog_posts WHERE published = 1 ORDER BY created_at DESC LIMIT 20"
  ).all();
  return results;
}
```

### 3. Blog Post Page вҖ” `/blog/[slug]`

**Route:** `src/app/blog/[slug]/page.tsx`

**What it does:**
- Queries D1: `SELECT * FROM blog_posts WHERE slug = ? AND published = 1`
- Renders the full blog post with:
  - Hero image (from R2 URL)
  - Title (h1)
  - Date + reading time
  - Tag pills
  - Full HTML content
  - CTA section at bottom (link to cybiqon.in services)

**SEO:**
- Generate `<title>` and `<meta description>` from the title and excerpt
- Add Open Graph tags (og:title, og:description, og:image) using the R2 image URL
- Add structured data (JSON-LD Article schema)

```typescript
// src/app/blog/[slug]/page.tsx
import { getRequestContext } from '@cloudflare/next-on-pages';

export const runtime = 'edge';

export async function generateMetadata({ params }) {
  const { env } = getRequestContext();
  const post = await env.DB.prepare(
    "SELECT title, excerpt, image_url FROM blog_posts WHERE slug = ? AND published = 1"
  ).bind(params.slug).first();

  return {
    title: `${post.title} | Cybiqon AI Solutions Blog`,
    description: post.excerpt,
    openGraph: {
      title: post.title,
      description: post.excerpt,
      images: [post.image_url],
      type: 'article',
    },
  };
}

export default async function BlogPost({ params }) {
  const { env } = getRequestContext();
  const post = await env.DB.prepare(
    "SELECT * FROM blog_posts WHERE slug = ? AND published = 1"
  ).bind(params.slug).first();

  if (!post) return notFound();

  return (
    // render full blog post
  );
}
```

### 4. Blog Post Content Rendering

The `content` field in D1 stores HTML. Render it safely:

```typescript
<article
  className="prose prose-lg max-w-none"
  dangerouslySetInnerHTML={{ __html: post.content }}
/>
```

Use Tailwind's `@tailwindcss/typography` plugin for the `prose` class вҖ” it handles heading styles, paragraph spacing, list formatting, etc.

```bash
npm install @tailwindcss/typography
```

Add to Tailwind config:
```javascript
plugins: [require('@tailwindcss/typography')]
```

## How Content Gets Into D1 + R2

This is handled by the **social media pipeline** in a separate repo (`/home/ubuntu/cybiqon/social-agent`). You don't need to build this вҖ” it's already being built on that side. Here's what happens:

1. The social agent generates a blog post from the same research used for social media
2. It uploads the image to R2 via Cloudflare API:
   ```
   PUT https://api.cloudflare.com/client/v4/accounts/{account_id}/r2/buckets/cybiqon-blog/objects/{filename}
   ```
3. It inserts the blog post into D1 via Cloudflare API:
   ```
   POST https://api.cloudflare.com/client/v4/accounts/{account_id}/d1/database/{db_id}/query
   ```
4. The Next.js blog page picks it up automatically on next page load
